# 프로세스 vs 프로그램


- **프로그램(Program)**: 저장장치(SSD/HDD)에 있는 **실행 전 코드 파일**
- **프로세스(Process)**: 그 프로그램이 실행되어 메모리에 올라간 **실행 중인 작업 단위**

---

## 1) 쉽게 이해하는 비유

- **프로그램**은 `레시피(요리법)`
- **프로세스**는 `실제로 요리 중인 상태`

같은 레시피로 여러 사람이 동시에 요리할 수 있듯이,  
같은 프로그램 하나로도 프로세스는 여러 개 생길 수 있습니다.

---

## 2) 실제 예시

### 1) 크롬(Chrome)

- `chrome.exe` 파일 자체는 **프로그램**
- 크롬을 실행한 뒤 탭을 여러 개 열면, 메모리에서 돌아가는 각 작업들은 **프로세스(들)**

즉:
- 프로그램: 정적인 파일
- 프로세스: CPU 시간을 받으며 동작하는 동적인 실행 주체

### 2) 메신저 앱 (카카오톡/디스코드 등)

- 설치된 메신저 앱 파일은 **프로그램**
- 앱을 켜서 로그인하고 메시지를 주고받는 순간, OS 관점에서는 **프로세스**가 생성되어 실행됨
- 백그라운드에서 알림을 받는 것도 실행 중 프로세스가 자원을 사용하고 있다는 의미

### 3) VS Code 두 개 실행

- `Code.exe`는 **프로그램**
- VS Code 창을 2개 띄우면 보통 **여러 프로세스**가 생성됨(메인 프로세스, 확장 프로세스 등)
- 같은 프로그램이라도 실행 인스턴스가 늘어나면 프로세스 수도 증가

### 4) 파이썬 스크립트 실행

- `hello.py` 같은 파일은 **프로그램(코드 파일)**
- `python hello.py` 명령을 실행하는 순간, Python 인터프리터 기반의 **프로세스**가 생성되어 코드가 실행됨
- 같은 스크립트를 터미널 2개에서 동시에 실행하면 프로세스도 2개가 됨

---

## 3) 차이 정리

| 구분 | 프로그램 | 프로세스 |
|---|---|---|
| 상태 | 정적(Static) | 동적(Dynamic) |
| 위치 | 디스크(파일) | 메모리(RAM) |
| 자원 사용 | 거의 없음 | CPU, 메모리, 파일, 소켓 등 사용 |
| 생명주기 | 파일로 계속 존재 가능 | 생성→실행→종료 |
| 개수 관계 | 1개 프로그램 | 여러 프로세스로 실행 가능 |

---

---

## 4) 프로그램 → 프로세스가 되는 흐름

- **프로그램**: 실행될 준비가 된 코드(파일)
- **프로세스**: 그 코드를 실제로 실행 중인 “인스턴스”

즉, 프로그램을 실행하면 OS가 메모리에 올리고 CPU 시간을 주면서 **프로세스**가 됨.

---

### 여러 프로세스가 가능한 이유

1. **같은 프로그램을 여러 번 실행할 수 있음**
2. **한 번 실행해도 내부적으로 여러 프로세스를 만들 수 있음**

---

### 예시 1) 같은 프로그램을 여러 번 실행
- `notepad.exe`는 프로그램(파일)
- 메모장을 2번 실행하면
  → **프로세스 2개** 생성

### 예시 2) 한 번 실행했는데도 여러 프로세스
- 크롬을 한 번 실행해도 탭/렌더링/확장 등을 분리해서
  → **여러 프로세스**가 동시에 동작


---

## 5) 면접 예상 질문 + 답변

### Q1. 프로그램과 프로세스의 차이를 설명해보세요.

**A.**  
프로그램은 디스크에 저장된 실행 파일이고, 프로세스는 그 프로그램이 실행되어 OS로부터 자원을 할당받은 실행 단위입니다. 프로그램은 정적이고, 프로세스는 실행 상태와 자원 정보를 가지는 동적인 개체입니다.

### Q2. 같은 프로그램을 두 번 실행하면 어떻게 되나요?

**A.**  
같은 프로그램 코드로부터 **서로 다른 프로세스 2개**가 생성됩니다. 코드 영역은 공유될 수 있지만, 각 프로세스는 독립적인 주소 공간(스택/힙 등)과 PID를 가지므로 기본적으로 서로 분리되어 동작합니다.

### Q3. 운영체제가 프로세스를 관리할 때 중요한 정보는 무엇인가요?

**A.**  
프로세스 상태(Ready/Running/Waiting), PID, 프로그램 카운터, 레지스터 값, 메모리 정보, 열린 파일 정보 등이 중요합니다. 이런 정보는 PCB(Process Control Block)에 저장되고, 문맥 교환(Context Switching) 시 활용됩니다.
