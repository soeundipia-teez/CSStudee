### 메모리?

- CPU가 직접 데이터를 처리할 수 있는 주기억장치(RAM)를 의미
- CPU가 처리할 데이터를 저장하고, 필요할 때 고속으로 접근할 수 있게 돕는 역할

### 메모리 관리 목표와 핵심 개념

- 메모리 사용을 최적화해서 **필요한 만큼 할당**하고, 사용이 끝나면 **즉시 회수**하는 것이 목표
- CPU의 논리 주소를 실제 RAM의 물리 주소로 연결하는 '주소 바인딩'이 핵심이며, MMU(Memory Management Unit) 하드웨어가 이를 처리

### 메모리 단편화 (Fragmentation)

메모리 공간이 작은 조각으로 나뉘어 있어, 사용 가능한 총 **메모리 공간은 충분**함에도 불구하고 **프로세스를 할당하지 못하는 상태**를 의미

메모리 관리 효율성을 떨어뜨리고 시스템 성능 저하의 원인

- **내부 단편화**: 할당된 메모리 블록이 프로세스보다 커서 내부 공간이 남는 현상
- **외부 단편화**: 남은 공간의 총합은 충분해도 연속되지 않아 할당이 불가능한 현상

## 메모리 관리 기법

### 1. 메모리 풀

- **정의**: 일정 크기의 메모리 블록을 미리 할당해두고 필요할 때마다 재사용하는 기법
- **오버헤드 감소**: 매번 시스템 콜을 통해 메모리를 할당/해제하는 비용을 줄여 실행 속도를 높임
- **단편화 해결**: 고정된 크기의 블록을 사용하여 **외부 단편화**를 근본적으로 방지
- **활용**: 게임 엔진의 객체 관리나 DB 커넥션 풀 등 고성능 저지연 환경에서 필수적

### 2. 가상 메모리

- **정의**: 물리 메모리의 한계를 극복하기 위해 **디스크 공간을 메모리처럼 활용**하여 프로세스에 거대한 가상 주소 공간을 제공
- **핵심 메커니즘**
    - **요구 페이징(Demand Paging)**: 당장 실행에 필요한 페이지만 메모리에 올리는 효율적 할당 방식
    - **페이지 테이블 (Page Table)**: 가상 주소와 물리 주소의 매핑 정보를 저장하는 표. CPU가 가상 주소로 요청하면 MMU가 이 표를 보고 실제 위치를 찾음
    - **페이지 폴트(Page Fault)**: 가상 메모리에는 주소가 있지만 물리 메모리에 데이터가 없을 때 발생하는 인터럽트. 이때 OS는 디스크에서 해당 데이터를 찾아 메모리에 올림
- **장점**
    - **메모리 용량 한계 극복**: 실제 RAM보다 큰 프로그램을 실행할 수 있어, 사용자 경험 향상
    - **프로세스 독립성 및 보안**: 각 프로세스는 독립된 가상 공간을 가지므로 서로의 데이터를 침범할 수 없음.
    - **메모리 효율성**: 필요한 부분만 메모리에 올리므로 입출력(I/O) 횟수가 줄어들고 더 많은 프로그램을 동시에 실행할 수 있음.
